\begin{abstract}
Machine learning models in production suffer from distribution drift, where evolving input features degrade model performance. Existing unsupervised detectors signal that drift occurred but not which features drifted or how severe the shift is. This paper proposes Enhanced Adversarial Drift Detection (EADD), extending the D3 discriminative detector with three contributions: (1) an Adaptive Sequential Permutation Test (ASPT) for statistically rigorous drift confirmation, reducing the permutation budget by 60--80\%, with prediction entropy emitted as a continuous diagnostic output (not evaluated standalone); (2) SHAP-based drift localisation and a Drift Severity Index (DSI) for feature-level diagnosis and severity quantification; and (3) experiments benchmarking against seven state-of-the-art unsupervised detectors on synthetic and real-world streams. On synthetic data, EADD is the only detector achieving full drift-type coverage (4/4) with zero false alarms. On real-world benchmarks, EADD detects drift on three of four annotated datasets, outperforming D3 on detection delay in all three detected cases, up to 8x faster on incremental drift types (InsectsIncrAbrupt MTD 33 vs. D3 272.5). SHAP attribution achieves perfect accuracy (P@k = R@k = 1.0) on controlled synthetic scenarios with independent features. On stable streams, EADD produces zero false alarms versus D3's 54.6 on autocorrelated streams at the matched threshold of 0.7; the reduction versus D3 at threshold 0.6 is statistically significant (p = 0.0101, Mann--Whitney U).
\end{abstract}

\section{Introduction}

Machine learning models deployed in production environments encounter evolving data distributions---collectively termed distribution drift---that can degrade model performance if left undetected~\cite{b1,b2}. Drift may manifest as changes in input features (covariate drift) or in the input--output relationship (concept drift); detecting such changes early is essential for maintaining reliable inference~\cite{b3,b4}.

The drift detection landscape spans three paradigms. \textit{Supervised methods} such as DDM~\cite{b8} monitor error rates but require timely ground-truth labels. \textit{Statistical methods} employ hypothesis tests such as the Kolmogorov--Smirnov test~\cite{b6,b7} but operate per feature, missing multivariate shifts. \textit{Discriminative methods} such as D3~\cite{b9} train classifiers to distinguish reference from current data, achieving state-of-the-art robustness in the comprehensive benchmark by Lukats et al.~\cite{b10}. However, all existing methods share a common limitation: they report only \textit{whether} drift occurred, not \textit{which} features drifted or \textit{how severe} the shift is.

D3~\cite{b9}, the most robust unsupervised detector in the Lukats et al.\ benchmark~\cite{b10}, exemplifies this limitation with two critical gaps. First, D3 applies no formal significance test---it relies on a fixed AUC threshold without statistical justification, making it vulnerable to false alarms on autocorrelated data. Second, D3 follows a ``detect-and-discard'' pattern: the trained classifier is discarded after computing AUC, providing no information about which features drove the shift. Recent work has partially addressed related gaps: TRIPODD~\cite{b28} enables feature-interaction hypothesis testing but requires pre-specifying interactions and has quadratic complexity; HDPD~\cite{b29} provides per-feature performance attributions but requires labels and operates post-hoc on collected batches.

In the \textit{academic research} literature on unsupervised streaming detectors, no single framework integrates (1)~statistically rigorous multivariate detection, (2)~identification of \textit{which} features caused the shift, and (3)~quantification of \textit{how severe} the drift is. Commercial monitoring platforms (e.g., Vertex AI~\cite{b20}, SageMaker Clarify~\cite{b21}) address aspects of this in production but operate in supervised or batch settings and are not designed for real-time streaming. Nor has any prior work systematically evaluated the accuracy of SHAP-based feature attribution within discriminative streaming drift detection.

EADD is therefore an \textit{integration contribution}: each component draws on established methods (Gandy~\cite{b30} for sequential testing, TreeSHAP~\cite{b24,b31} for attribution), but their combination within a single unsupervised streaming pipeline is non-trivial. The AUC gate must precede ASPT to bound overall Type~I error conservatively; TreeSHAP must be applied to a \textit{discriminative} rather than predictive classifier; and the severity metric must be defined dimensionality-invariant to handle varying feature counts across datasets.

This paper proposes \textbf{Enhanced Adversarial Drift Detection (EADD)}, addressing this gap with three contributions:
\begin{enumerate}
    \item Inspired by the statistical analysis of the classifier two-sample test~\cite{b19,b27}, we introduce an \textbf{Adaptive Sequential Permutation Test (ASPT)}~\cite{b30} that gates drift confirmation via a formal permutation test, achieving 60--80\% permutation savings while preserving Type~I error control; $H_{\text{pred}}$ is emitted as a continuous diagnostic output at every monitoring step (Section~III-B,C).
    \item Building on SHAP feature distribution analysis~\cite{b24}, we propose \textbf{SHAP-based drift localisation} and a \textbf{Drift Severity Index (DSI)} that combines detection confidence with attribution concentration into a continuous severity score, identifying which features caused drift and quantifying how severe the shift is (Section~III-D).
    \item We conduct \textbf{intensive experiments} benchmarking EADD against seven state-of-the-art unsupervised detectors on synthetic and real-world streams, evaluating detection coverage, false alarm control, and SHAP attribution accuracy (Section~IV).
\end{enumerate}

\section{Related Work}

\subsection{Drift Detection Methods}

Drift detection methods can be classified along two axes: label availability and detection mechanism~\cite{b1,b2}.

\textit{Supervised methods} monitor model error rates and require timely ground-truth labels. DDM~\cite{b8} tracks the error rate and its standard deviation; EDDM~\cite{b11} monitors inter-error distances, improving sensitivity to gradual drift. Both react only \textit{after} performance has degraded, limiting utility when labels are delayed~\cite{b1,b5}.

\textit{Unsupervised methods} operate without labels by directly monitoring data distributions. Statistical approaches include ADWIN~\cite{b12}, which dynamically adjusts sliding window sizes and can detect gradual drift, and KSWIN~\cite{b7}, which applies the Kolmogorov--Smirnov two-sample test over sliding windows for abrupt or sudden shifts. While statistically principled, these methods test each feature \textit{independently} (univariate), making them blind to multivariate distributional shifts and causing multiple testing problems in high-dimensional settings~\cite{b6}.

\textit{Discriminative approaches} frame drift detection as binary classification: an auxiliary classifier is trained to distinguish reference from current distributions~\cite{b9,b13,b14,b19}. This naturally captures multivariate drift in a single test. The most prominent discriminative detector, \textbf{D3}~\cite{b9}, trains a logistic regression classifier on reference vs.\ current windows and triggers drift when AUC exceeds a fixed threshold. Lukats et al.~\cite{b10} identified D3 as the most robust unsupervised detector across 29 real-world datasets. However, D3 has three weaknesses: (1)~its linear discriminator misses non-linear feature interactions, causing failure on gradual and incremental drift; (2)~it applies no formal significance test, relying on a fixed AUC threshold vulnerable to false alarms on autocorrelated data; and (3)~it discards the trained classifier after computing AUC, providing no feature-level attribution. Other benchmarked detectors exhibit characteristic limitations: CSDDM~\cite{b15} and IBDD~\cite{b16} achieve broad coverage but with elevated false alarm rates; OCDD~\cite{b17} and SPLL~\cite{b18} are effective for specific drift types but miss others; BNDM struggles with high-dimensional data; and UDetect failed on 45\% of scenarios~\cite{b10}.

\subsection{Feature Attribution in Discriminative Detection}

The \textit{classifier two-sample test} (C2ST)~\cite{b19} provides principled statistical foundations for discriminative detection: if a classifier reliably separates reference from current data (AUC $\gg 0.5$), the distributions differ significantly. E-C2ST~\cite{b27} strengthened statistical rigour via anytime-valid e-values. However, neither C2ST nor E-C2ST provides feature-level attribution or operates in a streaming setting. The practical need for such attribution was demonstrated at industrial scale by Pan et al.~\cite{b13} at Uber, Qian et al.~\cite{b14} for credit scoring, and by Google's SHAP-based feature monitoring in Vertex AI~\cite{b20}.

Recent academic work has advanced individual aspects: TRIPODD~\cite{b28} enables feature-interaction hypothesis testing, HDPD~\cite{b29} provides per-feature performance decomposition, and Hinder et al.~\cite{b32} formalised feature-wise drift definitions. However, no existing work systematically evaluates SHAP-based feature attribution accuracy for identifying drift-causing features within discriminative detection, nor integrates attribution with severity quantification in an unsupervised streaming pipeline.

\section{Proposed Methodology}

\subsection{Step 1: Adaptive Reference Windowing}

EADD maintains two windows: a \textit{reference window} $W_{ref}$ ($N_{ref}$ samples) and a \textit{current window} $W_{cur}$ ($N_{cur}$ samples). The reference window is updated by \textit{reservoir sampling}~\cite{b22}: each new sample $x_t$ replaces a random element in $W_{ref}$ with probability $N_{ref}/t$, which keeps an unbiased sample of historical data over long streams. The current window is a standard sliding buffer of recent observations. Monitoring is triggered every $M$ samples to balance latency and cost; after confirmed drift, $W_{ref}$ is reset to the new regime.

\subsection{Step 2: Discriminative Classification and Prediction Entropy}

Following the C2ST framework~\cite{b19}, EADD constructs a binary classification task: samples from $W_{ref}$ are labelled 0 (reference) and samples from $W_{cur}$ are labelled 1 (current). A LightGBM gradient boosting classifier~\cite{b23} is trained on this combined dataset via stratified 5-fold cross-validation, and the out-of-fold AUC-ROC is computed. AUC~$\approx$~0.5 indicates the classifier cannot distinguish the two windows (no drift); AUC~$\gg$~0.5 indicates the distributions are separable (drift suspected). We use LightGBM rather than D3's logistic regression~\cite{b9} because gradient boosting captures non-linear feature interactions, providing a more powerful discriminator for the non-linear and incremental drift types that defeat D3's linear classifier.

\textbf{Prediction Entropy.} Beyond the aggregate AUC score, EADD extracts additional diagnostic information from the classifier's per-sample predictions. We compute the normalised binary entropy of the discriminative classifier's predicted probabilities:
\begin{equation}
H_{\text{pred}} = \frac{-1}{n}\sum_{i=1}^{n}\left[\hat{p}_i\log\hat{p}_i + (1{-}\hat{p}_i)\log(1{-}\hat{p}_i)\right]
\label{eq:hpred}
\end{equation}
where $\hat{p}_i = f_\theta(x_i)$ is the probability that sample $i$ belongs to $W_{cur}$: $H_{\text{pred}} \approx \ln 2$ indicates maximum uncertainty (no drift); $H_{\text{pred}} \rightarrow 0$ indicates high classifier confidence (drift likely). $H_{\text{pred}}$ remains informative at every monitoring step---including steps where AUC $< \tau$ and no detection gate is triggered---providing a continuous sub-threshold trend signal; its standalone predictive value is qualitative in the present work.

\subsection{Step 3: Adaptive Sequential Permutation Test (ASPT)}

If AUC exceeds a threshold $\tau$, EADD applies an \textbf{adaptive sequential permutation test} inspired by Gandy~\cite{b30}. Instead of a fixed budget of $B$ permutations, ASPT monitors evidence accumulation and terminates early:
\begin{enumerate}
    \item For each permutation $i = 1, \ldots, B_{\max}$: shuffle labels, retrain, compute $\text{AUC}^{(i)}_{\text{perm}}$. Update count $c_i = \#\{j \leq i : \text{AUC}^{(j)}_{\text{perm}} \geq \text{AUC}_{\text{actual}}\}$.
    \item \textbf{Early reject $H_0$:} If $c_i = 0$, $i \geq B_{\min}$, and $1/(i+1) < \alpha$, terminate and confirm drift with $p = 1/(i+1)$.
    \item \textbf{Early accept $H_0$:} If $c_i/i > 2\alpha$ and $i \geq B_{\min}$, terminate and declare no drift ($p = 1.0$).
    \item \textbf{Terminal:} If $B_{\max}$ is reached, $p = (c_{B_{\max}}{+}1)/(B_{\max}{+}1)$.
\end{enumerate}
Drift is confirmed if $p < \alpha$. The two-stage gate is conservative: overall Type~I error $\leq \Pr(\text{AUC}{>}\tau \mid H_0) \times \alpha \leq \alpha$. Under $\alpha = 0.05$, early rejection requires $i \geq 20$ (since $1/21 \approx 0.048 < 0.05$); all confirmed detections in Experiments~1 and~3 terminated at exactly $i = 20$ (verified from logged \texttt{last\_aspt\_permutations}). Stable streams terminate at $i = B_{\min} = 10$, yielding 60--80\% savings vs.\ fixed-budget $B_{\max} = 50$.

\subsection{Step 4: Feature Attribution and Drift Severity Index}

Upon drift confirmation, EADD exploits the trained discriminative classifier---which D3~\cite{b9} would discard---to diagnose \textit{which} features drive the detected drift using SHAP (SHapley Additive exPlanations)~\cite{b24}.

\textbf{Feature Attribution.} EADD applies TreeSHAP~\cite{b24} to the trained LightGBM discriminator. Per-feature importances $\bar{\phi}_j = \frac{1}{n}\sum_{i=1}^{n}|\phi_j^{(i)}|$ are normalised into a contribution vector $\mathbf{s} = (s_1, \ldots, s_d)$, $s_j = \bar{\phi}_j / \sum_k \bar{\phi}_k$, identifying which features drive the shift and feeding the DSI below. SHAP interaction values~\cite{b31} ($\Phi \in \mathbb{R}^{d \times d}$) are computed as a supplementary diagnostic but are not evaluated in the present experiments; validation on correlated-drift scenarios is left to future work.

\textbf{Drift Severity Index (DSI).} The normalised contribution vector $\mathbf{s}$ computed from TreeSHAP directly feeds the DSI: a composite severity metric combining detection confidence with attribution concentration:
\begin{equation}
\text{DSI} = \text{AUC} \times \left(1 - H_{\text{norm}}(\mathbf{s})\right)
\label{eq:dsi}
\end{equation}
where $H_{\text{norm}}(\mathbf{s}) = -\sum_{j=1}^{d} s_j \log_2 s_j \,/\, \log_2 d$ is the normalised Shannon entropy~\cite{b34}. The $\log_2 d$ denominator ensures $H_{\text{norm}} \in [0, 1]$ regardless of dimensionality. DSI-based prescriptions map severity to recommended action. The tier boundaries below are proposed heuristics requiring domain-specific calibration; they are not empirically validated in the present work (all experiment DSI values fall in the Low tier):
\begin{itemize}
    \item \textbf{Critical (DSI $> 0.6$):} Concentrated drift $\rightarrow$ audit specific data pipeline.
    \item \textbf{Moderate ($0.3$--$0.6$):} Subset drift $\rightarrow$ investigate shared data source.
    \item \textbf{Low ($< 0.3$):} Diffuse shift $\rightarrow$ schedule full model retraining.
\end{itemize}

\subsection{Algorithm Summary}

\begin{algorithm}[htbp]
\caption{EADD Detection Cycle}
\label{alg:eadd}
\begin{algorithmic}[1]
\REQUIRE Stream $\{x_t\}$, $W_{ref}$, $W_{cur}$, threshold $\tau$, significance $\alpha$
\ENSURE Drift decision, $H_{\text{pred}}$, DSI, feature attribution, prescription
\STATE Update $W_{cur}$; update $W_{ref}$ via reservoir sampling~\cite{b22}
\STATE Train LightGBM on $W_{ref} \cup W_{cur}$ with labels $\{0,1\}$
\STATE Compute AUC via stratified 5-fold CV
\STATE Compute $H_{\text{pred}}$ via Eq.~(\ref{eq:hpred})
\IF{AUC $> \tau$}
    \STATE $p \gets$ ASPT($W_{ref}$, $W_{cur}$, AUC) \COMMENT{Adaptive early stopping}
    \IF{$p < \alpha$}
        \STATE Compute SHAP values via TreeSHAP; interaction matrix computed as diagnostic output (not evaluated here)
        \STATE Compute DSI via Eq.~(\ref{eq:dsi})
        \STATE Generate DSI-based prescription
        \RETURN \textsc{Drift}, $H_{\text{pred}}$, DSI, SHAP ranking, prescription
    \ENDIF
\ENDIF
\RETURN \textsc{No-Drift}, $H_{\text{pred}}$
\end{algorithmic}
\end{algorithm}

EADD is the only detector that achieved full drift-type coverage (4/4) with zero false alarms. BNDM, CSDDM, IBDD, and OCDD also achieved 4/4 coverage (through different discriminators) but with substantially higher false alarm costs (4.75, 5.50, 46.25, and 17.50 mean FA, respectively)---demonstrating that coverage and false alarm control are separable objectives. D3 and SPLL detected only 2/4 types. UDetect failed on all four types.

\textbf{False Alarm Robustness.} On four stable streams, EADD produced zero false alarms. D3 was vulnerable to autocorrelated data (87.4 false alarms at $\tau{=}0.6$; 54.6 at $\tau{=}0.7$); ASPT correctly identifies autocorrelation as non-significant ($p > 0.05$). A Mann--Whitney U test confirmed significantly fewer false alarms for EADD vs.\ D3 at $\tau{=}0.6$ ($p = 0.0101$, D3's benchmark default~\cite{b10}); at the matched $\tau{=}0.7$ operating point, the difference is not statistically significant ($p = 0.2266$), though EADD still produces zero false alarms vs.\ D3's 54.6 on autocorrelated streams.

\section{Conclusion}

This paper presented EADD, extending discriminative drift detection with (1) ASPT + $H_{\text{pred}}$ for statistically rigorous drift confirmation, (2) SHAP attribution and DSI for feature-level diagnosis and severity quantification, and (3) evaluation against seven state-of-the-art detectors on synthetic and real-world streams.

Evaluation demonstrated that EADD achieves full drift-type coverage (4/4) via LightGBM's non-linear discrimination, with zero false alarms via ASPT's significance filter---the only detector to achieve both simultaneously on synthetic benchmarks. EADD detects drift on three of four annotated real-world datasets, outperforming D3 on MTD in all three detected cases (InsectsAbrupt: 135 vs.\ 155.5; InsectsIncrAbrupt: 33 vs.\ 272.5, 8$\times$ faster; InsectsIncrRecur: 133 vs.\ 323.5; single-run estimates without variance). SHAP attribution achieves perfect accuracy (P@k\,{=}\,R@k\,{=}\,1.0) on controlled synthetic scenarios with independent features; the non-trivial result is univariate attribution (1/10 drifted features correctly identified). On stable streams, EADD produces zero false alarms vs.\ D3's 54.6 on autocorrelated streams at the matched $\tau{=}0.7$ operating point; the reduction vs.\ D3 at $\tau{=}0.6$ is statistically significant ($p = 0.0101$). By answering ``Is drift emerging?'', ``Is it significant?'', ``Which features caused it?'', and ``How severe?'', EADD transforms binary alarms into actionable diagnostics.

\subsection{Limitations and Future Work}

ASPT reduces the permutation budget by 60--80\% relative to fixed-budget ($B_{\max}{=}50$) permutation testing---a saving within EADD's own statistical stage. Separately, the discriminative paradigm overall remains ${\sim}50\times$ costlier than D3; this overhead arises from LightGBM training, not the permutation test, and is independent of the ASPT saving. No runtime table is reported. DSI ordering across scenarios is algebraically expected from its formula and does not constitute independent empirical validation; external ground-truth validity is not established---we do not show that higher DSI correlates with downstream model performance degradation. SHAP attributions are correlational, not causal~\cite{b33}. Attribution accuracy was tested on 3 controlled synthetic scenarios with independent features; performance under correlated or redundant features requires separate validation. No ablation isolates component contributions: replacing D3's logistic regression with LightGBM alone (without ASPT) may already achieve 4/4 coverage, so the benefit attributable to ASPT specifically is unknown. DSI tiers require domain-specific calibration. Prescriptions are heuristic and not evaluated by downstream outcomes (e.g., retraining quality). Future work: e-value testing~\cite{b27}, causal attribution~\cite{b33}, adversarial robustness.
