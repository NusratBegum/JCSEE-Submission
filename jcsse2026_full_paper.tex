% ===== ABSTRACT (~150 words) =====
\noindent\textbf{\textit{Abstract}}\textemdash\textit{Machine learning models in production suffer from distribution drift, where evolving input features degrade model performance. Existing unsupervised detectors signal that drift occurred but not which features drifted or how severe the shift is. This paper proposes Enhanced Adversarial Drift Detection (EADD), extending D3 with three contributions: (1) an Adaptive Sequential Permutation Test (ASPT) with prediction entropy monitoring ($H_{\text{pred}}$) for statistically rigorous drift confirmation with early-warning signals, (2) SHAP interaction-aware drift localisation with a Drift Severity Index (DSI) for feature-level diagnosis and severity quantification, and (3) intensive experiments validating detection coverage, false alarm control, and attribution accuracy. Evaluated against seven state-of-the-art unsupervised detectors on synthetic and 11 real-world benchmark datasets, EADD achieves zero false alarms on stable streams, perfect controlled-scenario feature attribution (P@k\,{=}\,R@k\,{=}\,1.0), and statistically significant false-alarm reduction versus D3 ($p{=}0.0101$). Under the default benchmark configuration EADD operates conservatively, prioritising precision over recall; operating-point recalibration for sensitivity-critical deployments is identified as future work.}

% ===== INTRODUCTION =====
\section{Introduction}

Machine learning models deployed in production environments encounter evolving data distributions. These changes---collectively termed distribution drift---can manifest as changes in input features (covariate drift) or in the input-output relationship (concept drift), causing performance degradation if left unmanaged~[1],~[2]. Detecting such changes early is essential for maintaining reliable inference~[3],~[4].

Conventional monitoring strategies react to drift only after observable performance degradation, which is inadequate in production contexts where ground-truth labels may be delayed or unavailable~[1],~[5]. The drift detection landscape has matured considerably: statistical methods employ hypothesis tests such as the Kolmogorov--Smirnov test~[6],~[7]; supervised detectors like DDM monitor error rates but require labels~[8]; and discriminative methods like D3 train classifiers to distinguish old from new data, achieving state-of-the-art accuracy in recent benchmarks~[9],~[10]. However, all existing methods share a common limitation: they report only \textit{whether} drift occurred, not \textit{which} features drifted or \textit{how severe} the shift is.

D3~[9] was identified as the most robust unsupervised detector in the Lukats et al.\ benchmark~[10], yet it has two critical operational gaps. First, D3 applies no formal significance test---it relies on a fixed AUC threshold of 0.7 without statistical justification, making it vulnerable to false alarms on autocorrelated or correlated data. Second, D3 follows a ``detect-and-discard'' pattern: the trained classifier is discarded after computing AUC, providing no information about \textit{which} features drove the detected drift. Without knowing which features drifted, an engineer cannot determine whether to audit a specific data pipeline, retrain the model, or investigate a data source. Recent work has partially addressed related gaps: TRIPODD~[28]\footnote{Feature-inTeraction awaRe InterPretable mOdel Drift Detection.} enables feature-interaction aware hypothesis testing but requires pre-specifying which interactions to test and has quadratic complexity in the number of features. HDPD~[29] provides per-feature performance attributions but is supervised (requires ground-truth labels) and operates post-hoc on collected batches, making it inapplicable when labels are delayed by days or weeks.

These partial solutions leave a critical operational gap. In production, distribution shifts are not isolated events: a feature may drift subtly for weeks before a statistical detector triggers, at which point the practitioner still does not know \textit{which} of potentially hundreds of features caused it, how severe the shift is, or whether to retrain the model, source more data, or flag a specific data pipeline. Manual diagnosis of individual features is impractical at scale: the time between a drift alarm and remediation (MTTD to MTTR) directly translates to degraded predictions and, in regulated domains such as healthcare or credit scoring, potential compliance violations under frameworks such as the EU AI Act~[3],~[4]. Production ML systems therefore require a single integrated pipeline that can: (1) detect drift with statistical rigour and computational efficiency, (2) identify \textit{which} specific features and feature interactions caused the shift, and (3) quantify \textit{how severe} the drift is relative to the baseline---all without ground-truth labels. Without this integration, practitioners must combine multiple disconnected tools, manually correlate their outputs, and interpret conflicting signals, significantly delaying remediation and increasing the risk of silent model degradation~[1],~[4].

This paper proposes \textbf{Enhanced Adversarial Drift Detection (EADD)}, addressing this gap with three contributions:
\begin{enumerate}
    \item Inspired by the statistical analysis of the classifier two-sample test~[19] and its extensions~[27], we introduce an \textbf{Adaptive Sequential Permutation Test (ASPT)}~[30] with accompanying \textbf{prediction entropy monitoring} ($H_{\text{pred}}$) that provides early-warning signals and reduces computational overhead by over 75\% while preserving Type~I error control (Section~III-B,C).
    \item Building on SHAP feature distribution analysis~[24], we propose \textbf{interaction-aware drift localisation}~[31] and a \textbf{Drift Severity Index (DSI)} that combines detection confidence with attribution concentration into a continuous severity score, identifying which features caused drift and quantifying how severe the shift is (Section~III-D).
    \item We conduct \textbf{intensive experiments} benchmarking EADD against seven state-of-the-art unsupervised detectors on synthetic and real-world streams, evaluating detection coverage, false alarm control, and SHAP attribution accuracy (Section~IV).
\end{enumerate}

% ===== RELATED WORK =====
\section{Related Work}

\subsection{Drift Detection Methods}

Drift detection methods can be classified along two axes: label availability and detection mechanism~[1],~[2].

\textit{Supervised methods} monitor model error rates and require timely ground-truth labels. DDM~[8] tracks the online error rate and its standard deviation, triggering drift alerts when these exceed binomial distribution bounds; it is effective for abrupt drift but slow to detect gradual changes. EDDM~[11] improves sensitivity to gradual drift by monitoring the distance between consecutive errors rather than the raw error rate. Both methods react to drift only \textit{after} performance has already degraded, limiting their utility in production settings where labels may be delayed by days or weeks~[1],~[5].

\textit{Unsupervised methods} operate without labels by directly monitoring data distributions. Statistical approaches include ADWIN~[12], which dynamically adjusts sliding window sizes and can detect gradual drift, and KSWIN~[7], which applies the Kolmogorov--Smirnov two-sample test over sliding windows for abrupt or sudden shifts. While statistically principled, these methods test each feature \textit{independently} (univariate), making them blind to multivariate distributional shifts and causing multiple testing problems in high-dimensional settings~[6].

\textit{Discriminative approaches} address this by training auxiliary classifiers to distinguish reference from current distributions---framing drift detection as binary classification~[9],~[13],~[14],~[19]. This naturally captures multivariate drift in a single unified test, regardless of drift type. The most prominent discriminative detector, \textbf{D3}~[9], trains a \textit{logistic regression} classifier on reference vs.\ current windows and triggers drift when the out-of-fold AUC exceeds a fixed threshold of 0.7. D3 was identified as the most robust unsupervised detector in the comprehensive real-world benchmark by Lukats et al.~[10]---it detects abrupt and recurring drift reliably, outperforming statistical methods on high-dimensional data. However, D3 has three critical weaknesses that limit its operational utility: (1)~its linear discriminator (logistic regression) misses non-linear feature interactions, causing it to fail on gradual and incremental drift patterns; (2)~it applies no formal significance test, relying on a fixed AUC threshold of 0.7 that is vulnerable to false alarms on autocorrelated or correlated data; and (3)~it discards the trained classifier after computing AUC---the ``detect-and-discard'' pattern---providing no information about \textit{which} features drove the detected drift. Other benchmarked detectors exhibit characteristic limitations: CSDDM~[15] and IBDD~[16] achieve broad coverage but suffer elevated false alarm rates; OCDD~[17] and SPLL~[18] are effective for specific drift types but miss others; BNDM struggles with small window sizes; and UDetect failed on 45\% of benchmark scenarios~[10].

\subsection{Feature Attribution in Discriminative Detection}

The practical need for feature-level attribution in discriminative drift detection was first demonstrated at industrial scale by Pan et al.~[13], who applied adversarial validation at Uber to diagnose train-deployment mismatches, and by Qian et al.~[14], who applied the same technique for credit scoring shift management. \textbf{D3}~[9] operationalised this approach for streaming detection: it trains a logistic regression classifier to separate reference from current windows and triggers drift when AUC exceeds a fixed threshold of 0.7. However, D3 applies no formal significance test---the fixed AUC threshold lacks statistical justification---making it vulnerable to false alarms on autocorrelated data. Despite strong detection performance, D3 and all streaming discriminative detectors discard the trained classifier after computing AUC, losing the diagnostic value embedded in the classifier's learned decision boundary. This ``detect-and-discard'' pattern means practitioners know drift occurred but receive no guidance on which features caused it.

The \textit{classifier two-sample test} (C2ST), formalised by Lopez-Paz and Oquab~[19], provides principled statistical foundations for this paradigm: reference data is labelled 0, current data is labelled 1; if a classifier reliably separates them (AUC $\gg 0.5$), the distributions differ significantly. E-C2ST~[27] further strengthened the statistical rigor of C2ST by replacing fixed-budget permutation tests with anytime-valid e-values. However, both C2ST and E-C2ST are proposed for distribution testing, not streaming drift detection---and neither provides feature-level attribution of the detected shift.

Industrial evidence validates the need for feature-level attribution in production systems: Google's Vertex AI uses SHAP-based feature attribution monitoring that detected degradation missed by aggregate input-distribution monitoring~[20], and Amazon SageMaker Clarify similarly integrates SHAP-based attribution for production drift detection~[21]. Recent academic work has advanced individual aspects: TRIPODD~[28] enables feature-interaction aware hypothesis testing, HDPD~[29] provides hierarchical performance decomposition with per-feature attributions, and Hinder et al.~[32] formalised rigorous mathematical definitions of feature-wise drift. However, no existing work systematically evaluates the accuracy of SHAP-based feature attribution for identifying which features cause drift within discriminative detection---nor integrates such attribution with severity quantification in an unsupervised streaming pipeline.

\section{Proposed Methodology}

As identified in Section~II, existing C2ST-based detectors lack early-warning signals, statistical significance testing, feature-level attribution, and severity quantification. EADD is a streaming pipeline for feature drift ($P(X)$ shift) detection that extends the C2ST framework~[19] with four novel components to address each of these gaps (Fig.~\ref{fig:pipeline}).

\subsection{Step 1: Adaptive Reference Windowing}

EADD maintains two windows: a \textit{reference window} $W_{ref}$ ($N_{ref}$ samples) and a \textit{current window} $W_{cur}$ ($N_{cur}$ samples). The reference window is updated by \textit{reservoir sampling}~[22]: each new sample $x_t$ replaces a random element in $W_{ref}$ with probability $N_{ref}/t$, which keeps an unbiased sample of historical data over long streams. The current window is a standard sliding buffer of recent observations. Monitoring is triggered every $M$ samples to balance latency and cost; after confirmed drift, $W_{ref}$ is reset to the new regime.

\subsection{Step 2: Discriminative Classification and Prediction Entropy}

Following the C2ST framework~[19], EADD constructs a binary classification task: samples from $W_{ref}$ are labelled 0 (reference) and samples from $W_{cur}$ are labelled 1 (current). A LightGBM gradient boosting classifier~[23] is trained on this combined dataset via stratified 5-fold cross-validation, and the out-of-fold AUC-ROC is computed. AUC~$\approx$~0.5 indicates the classifier cannot distinguish the two windows (no drift); AUC~$\gg$~0.5 indicates the distributions are separable (drift suspected). We use LightGBM rather than D3's logistic regression~[9] because gradient boosting captures non-linear feature interactions and has been shown to be a more powerful discriminator in two-sample testing~[19].

\textbf{Prediction Entropy.} Beyond the aggregate AUC score, EADD extracts additional diagnostic information from the classifier's per-sample predictions. We compute the normalised binary entropy of the discriminative classifier's predicted probabilities:
\begin{equation}
H_{\text{pred}} = \frac{-1}{n}\sum_{i=1}^{n}\left[\hat{p}_i\log\hat{p}_i + (1{-}\hat{p}_i)\log(1{-}\hat{p}_i)\right]
\label{eq:pui}
\end{equation}
where $\hat{p}_i = f_\theta(x_i)$ is the predicted probability that sample $i$ belongs to $W_{cur}$. $H_{\text{pred}} \approx \ln 2$ indicates maximum uncertainty (no drift); $H_{\text{pred}} \rightarrow 0$ indicates high classifier confidence (drift likely). A declining $H_{\text{pred}}$ trend across consecutive windows serves as an early-warning signal for emerging drift before statistical significance is reached.

\subsection{Step 3: Adaptive Sequential Permutation Test (ASPT)}

If AUC exceeds a threshold $\tau$, EADD applies an \textbf{adaptive sequential permutation test} inspired by Gandy~[30]. Instead of a fixed budget of $B$ permutations, ASPT monitors evidence accumulation and terminates early:
\begin{enumerate}
    \item For each permutation $i = 1, \ldots, B_{\max}$: shuffle labels, retrain, compute $\text{AUC}^{(i)}_{\text{perm}}$. Update count $c_i = \#\{j \leq i : \text{AUC}^{(j)}_{\text{perm}} \geq \text{AUC}_{\text{actual}}\}$.
    \item \textbf{Early reject $H_0$:} If $c_i = 0$, $i \geq B_{\min}$, and $1/(i+1) < \alpha$, terminate and confirm drift with $p = 1/(i+1)$.
    \item \textbf{Early accept $H_0$:} If $c_i/i > 2\alpha$ and $i \geq B_{\min}$, terminate and declare no drift ($p = 1.0$).
    \item \textbf{Terminal:} If $B_{\max}$ is reached, $p = (c_{B_{\max}}{+}1)/(B_{\max}{+}1)$.
\end{enumerate}
Drift is confirmed if $p < \alpha$. This preserves standard permutation-test interpretation under adaptive stopping by using either the terminal estimate or the conservative early-reject upper bound.

\subsection{Step 4: Feature Attribution and Drift Severity Index}

Upon drift confirmation, EADD exploits the trained discriminative classifier---which D3~[9] would discard---to diagnose \textit{which} features drive the detected drift using SHAP (SHapley Additive exPlanations)~[24].

\textbf{Feature Attribution.} EADD applies TreeSHAP~[24], a polynomial-time algorithm for computing exact Shapley values for tree-based models, to the trained LightGBM discriminator. For each sample, TreeSHAP produces a SHAP value $\phi_j$ for each feature $j \in \{1, \ldots, d\}$, indicating how much that feature contributed to the classifier's ability to distinguish reference from current data. \textit{Per-feature} importances (each feature's individual contribution, averaged across all samples) are computed as $\bar{\phi}_j = \frac{1}{n}\sum_{i=1}^{n}|\phi_j^{(i)}|$ and normalised into a contribution vector $\mathbf{s} = (s_1, \ldots, s_d)$ where $s_j = \bar{\phi}_j / \sum_k \bar{\phi}_k$. This vector $\mathbf{s}$ serves two purposes: (1)~it identifies which individual features drive the detected drift, and (2)~it feeds directly into the DSI computation below to quantify drift severity. Additionally, EADD computes \textbf{SHAP interaction values}~[31] via the interaction matrix $\Phi \in \mathbb{R}^{d \times d}$, where $\Phi_{ij}$ captures the joint effect of features $i$ and $j$ beyond their individual contributions. Significant interactions (exceeding $\mu + 2\sigma$ of the interaction distribution) are reported as supplementary diagnostics, addressing \textit{correlated feature drift}---where the joint distribution $P(X_i, X_j)$ shifts while individual feature distributions remain stable---a pattern that per-feature attribution alone misses~[32]. The per-feature contribution vector $\mathbf{s}$ is used for DSI severity quantification, while interaction diagnostics guide practitioners toward correlated root causes.

\textbf{Drift Severity Index (DSI).} The normalised contribution vector $\mathbf{s}$ computed from TreeSHAP directly feeds the DSI: a composite severity metric combining detection confidence with attribution concentration:
\begin{equation}
\text{DSI} = \text{AUC} \times \left(1 - H_{\text{norm}}(\mathbf{s})\right)
\label{eq:dsi}
\end{equation}
where $\mathbf{s} = (s_1, \ldots, s_d)$ is the normalised SHAP contribution vector (i.e., $s_j = \bar{\phi}_j / \sum_k \bar{\phi}_k$), $d$ is the number of input features, and $H_{\text{norm}}(\mathbf{s}) = -\sum_{j=1}^{d} s_j \log_2 s_j \,/\, \log_2 d$ is the normalised Shannon entropy~[34]. The $\log_2 d$ denominator ensures $H_{\text{norm}} \in [0, 1]$ regardless of dimensionality. Intuitively, when drift is concentrated in a single feature, $H_{\text{norm}} \approx 0$ and $\text{DSI} \approx \text{AUC}$ (high severity); when drift is uniformly spread across all features, $H_{\text{norm}} \approx 1$ and $\text{DSI} \approx 0$ (diffuse shift). DSI-based prescriptions replace heuristic thresholds:
\begin{itemize}
    \item \textbf{Critical (DSI $> 0.6$):} Concentrated drift $\rightarrow$ audit specific data pipeline.
    \item \textbf{Moderate ($0.3$--$0.6$):} Subset drift $\rightarrow$ investigate shared data source.
    \item \textbf{Low ($< 0.3$):} Diffuse shift $\rightarrow$ schedule full model retraining.
\end{itemize}

\subsection{Algorithm Summary}

\begin{algorithm}[htbp]
\caption{EADD Detection Cycle}
\label{alg:eadd}
\begin{algorithmic}[1]
\REQUIRE Stream $\{x_t\}$, $W_{ref}$, $W_{cur}$, threshold $\tau$, significance $\alpha$
\ENSURE Drift decision, $H_{\text{pred}}$, DSI, feature attribution, prescription
\STATE Update $W_{cur}$; update $W_{ref}$ via reservoir sampling~[22] (each new $x_t$ replaces a random element with probability $N_{ref}/t$, maintaining a uniform sample of history)
\STATE Train LightGBM on $W_{ref} \cup W_{cur}$ with labels $\{0,1\}$
\STATE Compute AUC via stratified 5-fold CV
\STATE Compute $H_{\text{pred}}$ via Eq.~(\ref{eq:pui})
\IF{AUC $> \tau$}
    \STATE $p \gets$ ASPT($W_{ref}$, $W_{cur}$, AUC) \COMMENT{Adaptive early stopping}
    \IF{$p < \alpha$}
        \STATE Compute SHAP values + interaction matrix via TreeSHAP
        \STATE Compute DSI via Eq.~(\ref{eq:dsi})
        \STATE Generate DSI-based prescription
        \RETURN \textsc{Drift}, $H_{\text{pred}}$, DSI, SHAP ranking, prescription
    \ENDIF
\ENDIF
\RETURN \textsc{No-Drift}, $H_{\text{pred}}$
\end{algorithmic}
\end{algorithm}

\subsection{Qualitative Component Analysis}

To validate each EADD component, we examine the behaviour of AUC and ASPT p-values on real-world streams from Experiment~2, providing qualitative insight into how each component contributes.

\textbf{AUC Trajectory.} On the INSECTS Abrupt Balanced stream, the discriminative AUC rises sharply from $\approx 0.51$ (no drift) to $\approx 0.78$ within 2 monitoring windows after the known drift point, confirming that the LightGBM discriminator detects the distributional shift. On the INSECTS Gradual Balanced stream, AUC increases slowly over multiple windows---a pattern that D3's fixed threshold misses early but EADD's $H_{\text{pred}}$ begins to flag as a declining entropy trend several windows before AUC exceeds $\tau = 0.7$. This demonstrates the early-warning value of $H_{\text{pred}}$: it provides a progressive signal when AUC alone is not yet decisive.

\textbf{ASPT p-value Trajectory.} On stable segments of the INSECTS streams (pre-drift), ASPT consistently returns $p > 0.05$, with early-accept termination (average $\approx 12$ permutations out of $B_{\max} = 50$). On the autocorrelated AR(1) stable stream, D3 triggers 54.6 false alarms at $\tau = 0.7$ because AUC fluctuates above the fixed threshold due to temporal correlation. EADD's ASPT correctly classifies these as non-significant ($p > 0.05$ in all runs), as the permutation test accounts for the magnitude of AUC not merely exceeding a threshold. After confirmed drift on SineClusters ($p = 0.019$), the ASPT reaches early-reject termination in $\approx 15$ permutations---a 70\% reduction versus the fixed $B = 50$ budget.

\textbf{Component Summary.} The LightGBM discriminator and reservoir sampling provide the detection breadth gains. ASPT provides Type~I error control that eliminates false alarms without requiring threshold recalibration. $H_{\text{pred}}$ provides an early-warning signal before statistical significance is reached. SHAP and DSI add attribution and severity outputs without affecting detection decisions.

% ===== CONCLUSION =====
\section{Conclusion}

This paper presented EADD (Enhanced Adversarial Drift Detection), a framework that advances discriminative drift detection through three methodological contributions: (1) an Adaptive Sequential Permutation Test with prediction entropy monitoring for statistically rigorous, computationally efficient drift confirmation, (2) SHAP interaction-aware feature attribution and a Drift Severity Index for feature-level diagnosis and severity quantification, and (3) intensive experimental validation against seven state-of-the-art detectors.

Evaluation demonstrated that EADD: (1)~achieves strong false-alarm control on stable data (0.0 mean false alarms), (2)~produces significantly fewer false alarms than D3 at $\tau{=}0.6$ ($p = 0.0101$), (3)~achieves perfect controlled-scenario feature attribution accuracy (P@k\,{=}\,R@k\,{=}\,1.0), and (4)~requires additional calibration to improve detection sensitivity in multi-detector benchmark settings. By answering ``Is drift emerging?'', ``Is it significant?'', and ``Which features caused it?'', EADD transforms binary drift alarms into actionable diagnostic reports.

\subsection{Limitations and Future Work}

ASPT reduces computational cost by over 75\% compared to fixed-budget permutation testing, but the adversarial paradigm remains ${\sim}50\times$ costlier than threshold-based D3. The DSI severity tiers ($0.3$, $0.6$) require domain-specific calibration. SHAP attributions are correlational, not causal~[33]. Future work includes e-value sequential testing~[27], causal drift attribution~[33], and adversarial robustness evaluation.
