% ===== ABSTRACT (~150 words) =====
\noindent\textbf{\textit{Abstract}}\textemdash\textit{Machine learning models in production suffer from distribution drift, where evolving input features degrade model performance. Existing unsupervised detectors signal that drift occurred but not which features drifted or what corrective action to take. This paper proposes Explainable Adversarial Drift Detection (EADD), extending discriminative classification with four novel contributions: (1) a Prediction Uncertainty Index (PUI) for entropy-based early warning, (2) an Adaptive Sequential Permutation Test (ASPT) that reduces computational overhead by up to 70\% while maintaining Type~I error control, (3) SHAP interaction-aware drift localisation for correlated feature diagnosis, and (4) a Drift Severity Index (DSI) for continuous severity quantification and automated remediation prescriptions. Evaluated against seven state-of-the-art detectors across synthetic and 11 real-world benchmarks, EADD uniquely achieved complete drift-type coverage (4/4) with near-zero false alarms while providing perfect feature-level attribution (P@k\,{=}\,R@k\,{=}\,1.0).}

% ===== INTRODUCTION =====
\section{Introduction}

Machine learning models deployed in production environments encounter evolving data distributions. These changes---collectively termed distribution drift---can manifest as changes in input features (covariate drift) or in the input-output relationship (concept drift), causing performance degradation if left unmanaged~[1],~[2]. Detecting such changes early is essential for maintaining reliable inference~[3],~[4].

Conventional monitoring strategies react to drift only after observable performance degradation, which is inadequate in production contexts where ground-truth labels may be delayed or unavailable~[1],~[5]. The drift detection landscape has matured considerably: statistical methods employ hypothesis tests such as the Kolmogorov--Smirnov test~[6],~[7]; supervised detectors like DDM monitor error rates but require labels~[8]; and discriminative methods like D3 train classifiers to distinguish old from new data, achieving state-of-the-art accuracy in recent benchmarks~[9],~[10]. However, all existing methods share a common limitation: they report only \textit{whether} drift occurred, not \textit{which} features drifted or \textit{what} corrective action is appropriate.

Recent advances have begun addressing individual aspects of this gap, but each solves only one dimension. E-C2ST~[27] replaces fixed-budget permutation tests with anytime-valid e-values for sequential testing, but provides no feature-level attribution---practitioners still do not know \textit{which} features drifted. TRIPODD~[28]\footnote{Feature-inTeraction awaRe InterPretable mOdel Drift Detection.} introduces feature-interaction aware hypothesis testing that can identify \textit{which} feature pairs drive drift, but relies on computationally expensive conditional independence tests and does not provide severity quantification or remediation guidance. HDPD~[29] decomposes performance gaps into per-feature attributions with nonparametric guarantees, but requires labelled data and operates post-hoc rather than in a streaming pipeline.

These partial solutions leave a critical operational gap: production ML systems need a single pipeline that can (1) detect drift with statistical rigour, (2) explain \textit{which} features and feature interactions caused it, (3) quantify \textit{how severe} the drift is, and (4) recommend \textit{what action} to take---all without requiring ground-truth labels and with computational costs practical for real-time monitoring. Without this integration, practitioners must manually combine multiple tools and interpret conflicting signals, delaying remediation and increasing the risk of silent model degradation~[1],~[4].

This paper proposes \textbf{Explainable Adversarial Drift Detection (EADD)}, addressing this gap with four contributions:
\begin{enumerate}
    \item A \textbf{Prediction Uncertainty Index (PUI)} derived from classifier entropy, providing early-warning signals for emerging distributional instabilities.
    \item An \textbf{Adaptive Sequential Permutation Test (ASPT)}~[30] that reduces computational overhead by up to 70\% while maintaining Type~I error control.
    \item \textbf{SHAP interaction-aware drift localisation}~[31] that identifies correlated feature drift invisible to marginal attribution.
    \item A \textbf{Drift Severity Index (DSI)} for continuous severity quantification and automated, proportional remediation prescriptions.
\end{enumerate}

The remainder of this paper is organized as follows: Section~II reviews related work. Section~III describes the EADD methodology. Section~IV presents experimental evaluation. Section~V concludes.

% ===== RELATED WORK =====
\section{Related Work}

\subsection{Drift Detection Methods}

Drift detection methods can be classified along two axes: label availability and detection mechanism~[1],~[2].

\textit{Supervised methods} monitor model error rates and require timely ground-truth labels. DDM~[8] tracks the online error rate and its standard deviation, triggering drift alerts when these statistics exceed binomial distribution bounds. EDDM~[11] extends DDM by monitoring the distance between consecutive errors, improving sensitivity to gradual drift. Both methods react to drift only \textit{after} performance has already degraded, limiting their utility in production settings where labels may be delayed by days or weeks~[1],~[5].

\textit{Unsupervised methods} operate without labels by directly monitoring data distributions. Statistical approaches include ADWIN~[12], which dynamically adjusts sliding window sizes using statistical change tests, and KSWIN~[7], which applies the Kolmogorov--Smirnov two-sample test over sliding windows. While statistically principled, these methods test each feature independently (univariate), leading to multiple testing problems in high-dimensional settings~[6]. Discriminative approaches instead train auxiliary classifiers to distinguish between reference and current distributions, framing drift detection as binary classification~[13],~[14],~[19]. This naturally handles multivariate drift in a single unified test.

A comprehensive benchmark by Lukats et al.~[10] evaluated numerous unsupervised detectors on real-world streams and identified D3~[9]---a discriminative method using logistic regression with a fixed AUC threshold of 0.7---as the most robust overall. However, D3 provides no feature-level diagnostics: it reports \textit{that} drift occurred but not \textit{which} features drifted. Furthermore, D3 uses a fixed threshold without statistical significance testing, making it vulnerable to false alarms on correlated or autocorrelated data. Other benchmarked detectors exhibit characteristic limitations: CSDDM~[15] (PCA + K-Means clustering) and IBDD~[16] (image-based comparison) achieve high coverage but suffer elevated false alarm rates; OCDD~[17] (one-class SVM) and SPLL~[18] (semi-parametric log-likelihood) are effective for specific drift types but miss others; BNDM (Bayesian non-parametric) struggles with small window sizes; and UDetect (Shewhart control charts) failed on 45\% of benchmark scenarios~[10].

\subsection{Classifier Two-Sample Tests and Explainability}

The \textit{classifier two-sample test} (C2ST), formalised by Lopez-Paz and Oquab~[19], frames distribution comparison as binary classification: data from a reference period is labelled 0 and data from the current period is labelled 1; if a classifier can reliably separate them (AUC $\gg 0.5$), the distributions differ significantly. This approach has been adopted at Uber for detecting train-deployment mismatch~[13] and for credit scoring shift management~[14]. D3~[9] applies this principle in streaming settings using logistic regression as the discriminator with a fixed AUC threshold---effectively a C2ST without formal significance testing.

Despite strong detection accuracy, existing C2ST-based detectors---including D3---discard the trained classifier after computing AUC, losing information about \textit{which} features differentiate the distributions. This ``detect-and-discard'' pattern wastes the diagnostic value embedded in the classifier's learned decision boundary~[19]. Industrial evidence validates the need for feature-level attribution: Google's Vertex AI uses feature attribution monitoring based on SHAP values~[24] that detected degradation missed by aggregate input-distribution monitoring~[20]. Amazon SageMaker Clarify similarly integrates SHAP-based attribution monitoring for production drift detection~[21]. Recent academic work has advanced individual aspects: TRIPODD~[28] enables feature-interaction aware hypothesis testing, HDPD~[29] provides hierarchical performance decomposition with per-feature attributions, and Hinder et al.~[32] formalised rigorous mathematical definitions of feature-wise drift. However, no existing framework unifies these capabilities into a single streaming detection-and-diagnosis pipeline that provides statistically calibrated detection, interaction-aware attribution, severity quantification, and automated remediation prescriptions.

\section{Proposed Methodology}

As identified in Section~II, existing C2ST-based detectors lack early-warning signals, statistical significance testing, feature-level attribution, and severity quantification. EADD is a streaming pipeline for feature drift ($P(X)$ shift) detection that extends the C2ST framework~[19] with four novel components to address each of these gaps (Fig.~\ref{fig:pipeline}).

\subsection{Step 1: Adaptive Reference Windowing}

EADD maintains two data windows. The \textit{reference window} $W_{ref}$ ($N_{ref}{=}500$ samples) represents the baseline distribution and is maintained via \textit{reservoir sampling}~[22]: each new sample $x_t$ replaces a randomly chosen element in $W_{ref}$ with probability $N_{ref}/t$, ensuring that all past samples have equal probability of representation regardless of stream length. This is critical because a simple sliding reference window would only capture the most recent history, not the global pre-drift distribution. The \textit{current window} $W_{cur}$ ($N_{cur}{=}200$ samples) is a standard sliding window containing the most recent observations. Monitoring is triggered every 50 samples to balance detection latency with computational cost. Upon confirmed drift, $W_{ref}$ is reset to the current data to establish a new baseline.

\subsection{Step 2: Discriminative Classification and PUI}

Following the C2ST framework~[19], EADD constructs a binary classification task: samples from $W_{ref}$ are labelled 0 (reference) and samples from $W_{cur}$ are labelled 1 (current). A LightGBM gradient boosting classifier~[23] is trained on this combined dataset via stratified 5-fold cross-validation, and the out-of-fold AUC-ROC is computed. AUC~$\approx$~0.5 indicates the classifier cannot distinguish the two windows (no drift); AUC~$\gg$~0.5 indicates the distributions are separable (drift suspected). We use LightGBM rather than D3's logistic regression~[9] because gradient boosting captures non-linear feature interactions and has been shown to be a more powerful discriminator in two-sample testing~[19].

\textbf{Prediction Uncertainty Index (PUI).} Beyond the aggregate AUC score, EADD extracts additional diagnostic information from the classifier's per-sample predictions. The PUI is defined as the normalised binary entropy of the discriminative classifier's predicted probabilities:
\begin{equation}
\text{PUI} = \frac{-1}{n}\sum_{i=1}^{n}\left[\hat{p}_i\log\hat{p}_i + (1{-}\hat{p}_i)\log(1{-}\hat{p}_i)\right]
\label{eq:pui}
\end{equation}
where $\hat{p}_i = f_\theta(x_i)$ is the predicted probability that sample $i$ belongs to $W_{cur}$. PUI~$\approx$~$\ln 2$ indicates maximum uncertainty (no drift); PUI~$\rightarrow$~0 indicates high classifier confidence (drift likely). A declining PUI trend across consecutive windows serves as an early-warning signal for emerging drift before statistical significance is reached.

\subsection{Step 3: Adaptive Sequential Permutation Test (ASPT)}

If AUC exceeds 0.7, EADD applies an \textbf{adaptive sequential permutation test} inspired by Gandy~[30]. Instead of a fixed budget of $B$ permutations, ASPT monitors evidence accumulation and terminates early:
\begin{enumerate}
    \item For each permutation $i = 1, \ldots, B_{\max}{=}50$: shuffle labels, retrain, compute $\text{AUC}^{(i)}_{\text{perm}}$. Update count $c_i = \#\{j \leq i : \text{AUC}^{(j)}_{\text{perm}} \geq \text{AUC}_{\text{actual}}\}$.
    \item \textbf{Early reject $H_0$:} If $c_i = 0$ and $i \geq 10$ and $(1{-}\alpha)^i < 0.05$, terminate and confirm drift ($p = 0.001$).
    \item \textbf{Early accept $H_0$:} If $c_i/i > 2\alpha$ and $i \geq 10$, terminate and declare no drift ($p = 1.0$).
    \item \textbf{Terminal:} If $B_{\max}$ is reached, $p = (c_{B_{\max}}{+}1)/(B_{\max}{+}1)$.
\end{enumerate}
Drift is confirmed if $p < \alpha{=}0.01$. In our experiments (Section~IV), we logged the number of permutations executed per ASPT invocation: the mean was 12.3 permutations when drift was present (early rejection via Rule~2) and 9.7 when absent (early acceptance via Rule~3), representing a 75--81\% reduction versus fixed $B{=}50$. This is consistent with the theoretical analysis of sequential Monte Carlo tests by Gandy~[30], who proved that early stopping maintains Type~I error control when termination rules satisfy the bounded resampling risk condition.

\subsection{Step 4: SHAP Interactions and Drift Severity Index}

Upon drift confirmation, EADD exploits the trained discriminative classifier---which D3~[9] would discard---to diagnose \textit{which} features drive the detected drift using SHAP (SHapley Additive exPlanations)~[24].

\textbf{Marginal and Interaction Attribution.} EADD applies TreeSHAP~[24], a polynomial-time algorithm for computing exact Shapley values for tree-based models, to the trained LightGBM discriminator. For each sample, TreeSHAP produces a vector of SHAP values $\phi_j$ for each feature $j \in \{1, \ldots, d\}$ (where $d$ is the number of input features), indicating how much that feature contributed to the classifier's ability to distinguish reference from current data. Marginal SHAP importances are computed as $\bar{\phi}_j = \frac{1}{n}\sum_{i=1}^{n}|\phi_j^{(i)}|$ and normalised to percentages. Additionally, EADD computes \textbf{SHAP interaction values}~[31] via the interaction matrix $\Phi \in \mathbb{R}^{d \times d}$, where $\Phi_{ij}$ captures the joint effect of features $i$ and $j$ beyond their individual contributions. Significant interactions (exceeding $\mu + 2\sigma$ of the interaction distribution) are reported alongside marginal contributions. This addresses \textit{correlated feature drift}---where the joint distribution $P(X_i, X_j)$ shifts while the marginals $P(X_i)$ and $P(X_j)$ remain stable---a pattern that marginal-only attribution methods miss~[32].

\textbf{Drift Severity Index (DSI).} EADD introduces a composite metric combining detection confidence with attribution concentration:
\begin{equation}
\text{DSI} = \text{AUC} \times \left(1 - H_{\text{norm}}(\mathbf{s})\right)
\label{eq:dsi}
\end{equation}
where $\mathbf{s} = (s_1, \ldots, s_d)$ is the normalised SHAP contribution vector (i.e., $s_j = \bar{\phi}_j / \sum_k \bar{\phi}_k$), $d$ is the number of input features, and $H_{\text{norm}}(\mathbf{s}) = -\sum_{j=1}^{d} s_j \log_2 s_j \,/\, \log_2 d$ is the normalised Shannon entropy~[34]. The $\log_2 d$ denominator ensures $H_{\text{norm}} \in [0, 1]$ regardless of dimensionality. Intuitively, when drift is concentrated in a single feature, $H_{\text{norm}} \approx 0$ and $\text{DSI} \approx \text{AUC}$ (high severity); when drift is uniformly spread across all features, $H_{\text{norm}} \approx 1$ and $\text{DSI} \approx 0$ (diffuse shift). DSI-based prescriptions replace heuristic thresholds:
\begin{itemize}
    \item \textbf{Critical (DSI $> 0.6$):} Concentrated drift $\rightarrow$ audit specific data pipeline.
    \item \textbf{Moderate ($0.3$--$0.6$):} Subset drift $\rightarrow$ investigate shared data source.
    \item \textbf{Low ($< 0.3$):} Diffuse shift $\rightarrow$ schedule full model retraining.
\end{itemize}

\subsection{Algorithm Summary}

\begin{algorithm}[htbp]
\caption{EADD Detection Cycle}
\label{alg:eadd}
\begin{algorithmic}[1]
\REQUIRE Stream $\{x_t\}$, $W_{ref}$, $W_{cur}$, threshold $\tau{=}0.7$
\ENSURE Drift decision, PUI, DSI, feature attribution, prescription
\STATE Update $W_{cur}$; update $W_{ref}$ via reservoir sampling
\STATE Train LightGBM on $W_{ref} \cup W_{cur}$ with labels $\{0,1\}$
\STATE Compute AUC via stratified 5-fold CV
\STATE Compute PUI via Eq.~(\ref{eq:pui})
\IF{AUC $> \tau$}
    \STATE $p \gets$ ASPT($W_{ref}$, $W_{cur}$, AUC) \COMMENT{Adaptive early stopping}
    \IF{$p < 0.01$}
        \STATE Compute SHAP values + interaction matrix via TreeSHAP
        \STATE Compute DSI via Eq.~(\ref{eq:dsi})
        \STATE Generate DSI-based prescription
        \RETURN \textsc{Drift}, PUI, DSI, SHAP ranking, prescription
    \ENDIF
\ENDIF
\RETURN \textsc{No-Drift}, PUI
\end{algorithmic}
\end{algorithm}

Following the evaluation protocol of Lukats et al.~[10], EADD was benchmarked against seven state-of-the-art unsupervised detectors: D3~[9], BNDM, CSDDM~[15], IBDD~[16], OCDD~[17], SPLL~[18], and UDetect.

\textbf{Experiment 1 (Temporal Patterns):} Synthetic streams ($n{=}10{,}000$, $d{=}5$) with controlled abrupt, gradual, incremental, and recurring drift injected at $t{=}5{,}000$. All eight detectors were compared. Five independent runs per drift type.

\textbf{Experiment 2 (Real-World Benchmark):} 11 real-world datasets from the USP Data Stream Repository~[25] following the Lukats et al.~[10] evaluation protocol: datasets with known drift points are evaluated by Mean Time to Detection (MTD) and Missed Drift Rate (MDR); datasets without ground truth are evaluated by total number of detections (selectivity).

\textbf{Experiment 3 (Feature Attribution):} Three controlled scenarios with $d{=}10$ features and drift at $t{=}5{,}000$: (a)~univariate drift in F3 only (mean shift $+2\sigma$), (b)~subset drift in F2/F5/F7, (c)~multivariate drift in all features. Measured Precision@k (P@k), Recall@k (R@k), and F1@k, where $k$ is the number of truly drifted features.

\textbf{Experiment 4 (False Alarms):} Four stable synthetic streams (Gaussian i.i.d., autocorrelated AR(1) with $\phi{=}0.8$, heteroscedastic with time-varying variance, correlated with $\rho{=}0.7$) containing zero drift. Five runs per stream type.

\textbf{Configuration.} EADD: LightGBM~[23] (30 estimators, max depth 6, learning rate 0.1), $N_{ref}{=}500$, $N_{cur}{=}200$, monitoring frequency 50, ASPT ($B_{\max}{=}50$, $B_{\min}{=}10$, $\alpha{=}0.01$, AUC threshold $\tau{=}0.7$). Random seeds fixed for reproducibility.

% ===== CONCLUSION =====
\section{Conclusion}

This paper presented EADD, a framework that advances discriminative drift detection through four methodological contributions: the Prediction Uncertainty Index for entropy-based early warning, Adaptive Sequential Permutation Testing for computationally efficient significance calibration, SHAP interaction-aware drift localisation for correlated feature diagnosis, and the Drift Severity Index for continuous severity quantification with automated prescriptions.

Evaluation against seven state-of-the-art detectors demonstrated that EADD: (1)~is the only detector achieving complete drift coverage (4/4 types) with near-zero false alarms (mean 0.25), (2)~achieves 44.7\% faster detection on WaveformDrift2 with zero false alarms, (3)~achieves perfect feature attribution accuracy (P@k\,{=}\,R@k\,{=}\,1.0) with 100\% prescription accuracy, and (4)~produces significantly fewer false alarms than D3 ($p = 0.013$). By answering ``Is drift emerging?'', ``Is it significant?'', ``What caused it?'', and ``How severe is it?'', EADD transforms binary drift alarms into actionable diagnostic reports.

\subsection{Limitations and Future Work}

ASPT reduces computational cost by 70\% compared to fixed-budget permutation testing, but the adversarial paradigm remains ${\sim}50\times$ costlier than threshold-based D3. The DSI severity tiers ($0.3$, $0.6$) require domain-specific calibration. SHAP attributions are correlational, not causal~[33]. Future work includes e-value sequential testing~[27], causal drift attribution~[33], and adversarial robustness evaluation.
