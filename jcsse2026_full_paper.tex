% ===== ABSTRACT (~150 words) =====
\noindent\textbf{\textit{Abstract}}\textemdash\textit{Machine learning models in production suffer from distribution drift, where evolving input features degrade model performance. Existing unsupervised detectors signal that drift occurred but not which features drifted or how severe the shift is. This paper proposes Enhanced Adversarial Drift Detection (EADD), extending the D3 discriminative detector with three contributions: (1) an Adaptive Sequential Permutation Test (ASPT) with prediction entropy monitoring for statistically rigorous drift confirmation, (2) SHAP interaction-aware drift localisation with a novel Drift Severity Index (DSI) for feature-level diagnosis and severity quantification, and (3) comprehensive experiments benchmarking against seven state-of-the-art unsupervised detectors on synthetic and real-world streams. On synthetic data, EADD is the only detector to achieve full drift-type coverage (4/4) with zero false alarms. On real-world benchmarks, EADD detects drift on three of four annotated datasets, outperforming D3 on detection delay. EADD further achieves perfect feature attribution accuracy (P@k\,{=}\,R@k\,{=}\,1.0) on controlled scenarios and statistically significant false-alarm reduction versus D3 ($p{=}0.0101$).}

% ===== INTRODUCTION =====
\section{Introduction}

Machine learning models deployed in production environments encounter evolving data distributions---collectively termed distribution drift---that can degrade model performance if left undetected~[1],~[2]. Drift may manifest as changes in input features (covariate drift) or in the input--output relationship (concept drift); detecting such changes early is essential for maintaining reliable inference~[3],~[4].

The drift detection landscape spans three paradigms. \textit{Supervised methods} such as DDM~[8] monitor error rates but require timely ground-truth labels. \textit{Statistical methods} employ hypothesis tests such as the Kolmogorov--Smirnov test~[6],~[7] but operate per feature, missing multivariate shifts. \textit{Discriminative methods} such as D3~[9] train classifiers to distinguish reference from current data, achieving state-of-the-art robustness in the comprehensive benchmark by Lukats et al.~[10],~[26]. However, all existing methods share a common limitation: they report only \textit{whether} drift occurred, not \textit{which} features drifted or \textit{how severe} the shift is.

D3~[9], the most robust unsupervised detector in the Lukats et al.\ benchmark~[10], exemplifies this limitation with two critical gaps. First, D3 applies no formal significance test---it relies on a fixed AUC threshold without statistical justification, making it vulnerable to false alarms on autocorrelated data. Second, D3 follows a ``detect-and-discard'' pattern: the trained classifier is discarded after computing AUC, providing no information about which features drove the shift. Recent work has partially addressed related gaps: TRIPODD~[28] enables feature-interaction hypothesis testing but requires pre-specifying interactions and has quadratic complexity; HDPD~[29] provides per-feature performance attributions but requires labels and operates post-hoc on collected batches.

No existing work provides a single unsupervised streaming pipeline that (1)~detects drift with statistical rigour, (2)~identifies \textit{which} features caused the shift, and (3)~quantifies \textit{how severe} the drift is. Nor has any work systematically evaluated the accuracy of SHAP-based feature attribution within discriminative drift detection.

This paper proposes \textbf{Enhanced Adversarial Drift Detection (EADD)}, addressing this gap with three contributions:
\begin{enumerate}
    \item Inspired by the statistical analysis of the classifier two-sample test~[19] and its extensions~[27], we introduce an \textbf{Adaptive Sequential Permutation Test (ASPT)}~[30] with accompanying \textbf{prediction entropy monitoring} ($H_{\text{pred}}$) that provides early-warning signals and reduces computational overhead by over 75\% while preserving Type~I error control (Section~III-B,C).
    \item Building on SHAP feature distribution analysis~[24], we propose \textbf{interaction-aware drift localisation}~[31] and a \textbf{Drift Severity Index (DSI)} that combines detection confidence with attribution concentration into a continuous severity score, identifying which features caused drift and quantifying how severe the shift is (Section~III-D).
    \item We conduct \textbf{intensive experiments} benchmarking EADD against seven state-of-the-art unsupervised detectors on synthetic and real-world streams, evaluating detection coverage, false alarm control, and SHAP attribution accuracy (Section~IV).
\end{enumerate}

% ===== RELATED WORK =====
\section{Related Work}

\subsection{Drift Detection Methods}

Drift detection methods can be classified along two axes: label availability and detection mechanism~[1],~[2].

\textit{Supervised methods} monitor model error rates and require timely ground-truth labels. DDM~[8] tracks the online error rate and its standard deviation, triggering drift alerts when these exceed binomial distribution bounds; it is effective for abrupt drift but slow to detect gradual changes. EDDM~[11] improves sensitivity to gradual drift by monitoring the distance between consecutive errors rather than the raw error rate. Both methods react to drift only \textit{after} performance has already degraded, limiting their utility in production settings where labels may be delayed by days or weeks~[1],~[5].

\textit{Unsupervised methods} operate without labels by directly monitoring data distributions. Statistical approaches include ADWIN~[12], which dynamically adjusts sliding window sizes and can detect gradual drift, and KSWIN~[7], which applies the Kolmogorov--Smirnov two-sample test over sliding windows for abrupt or sudden shifts. While statistically principled, these methods test each feature \textit{independently} (univariate), making them blind to multivariate distributional shifts and causing multiple testing problems in high-dimensional settings~[6].

\textit{Discriminative approaches} frame drift detection as binary classification: an auxiliary classifier is trained to distinguish reference from current distributions~[9],~[13],~[14],~[19]. This naturally captures multivariate drift in a single test. The most prominent discriminative detector, \textbf{D3}~[9], trains a logistic regression classifier on reference vs.\ current windows and triggers drift when AUC exceeds a fixed threshold. Lukats et al.~[10] identified D3 as the most robust unsupervised detector across 29 real-world datasets. However, D3 has three weaknesses: (1)~its linear discriminator misses non-linear feature interactions, causing failure on gradual and incremental drift; (2)~it applies no formal significance test, relying on a fixed AUC threshold vulnerable to false alarms on autocorrelated data; and (3)~it discards the trained classifier after computing AUC, providing no feature-level attribution. Other benchmarked detectors exhibit characteristic limitations: CSDDM~[15] and IBDD~[16] achieve broad coverage but with elevated false alarm rates; OCDD~[17] and SPLL~[18] are effective for specific drift types but miss others; BNDM struggles with high-dimensional data; and UDetect failed on 45\% of scenarios~[10].

\subsection{Feature Attribution in Discriminative Detection}

The \textit{classifier two-sample test} (C2ST)~[19] provides principled statistical foundations for discriminative detection: if a classifier reliably separates reference from current data (AUC $\gg 0.5$), the distributions differ significantly. E-C2ST~[27] strengthened statistical rigour via anytime-valid e-values. However, neither C2ST nor E-C2ST provides feature-level attribution or operates in a streaming setting. The practical need for such attribution was demonstrated at industrial scale by Pan et al.~[13] at Uber, Qian et al.~[14] for credit scoring, and by Google's SHAP-based feature monitoring in Vertex AI~[20].

Recent academic work has advanced individual aspects: TRIPODD~[28] enables feature-interaction hypothesis testing, HDPD~[29] provides per-feature performance decomposition, and Hinder et al.~[32] formalised feature-wise drift definitions. However, no existing work systematically evaluates SHAP-based feature attribution accuracy for identifying drift-causing features within discriminative detection, nor integrates attribution with severity quantification in an unsupervised streaming pipeline.

\subsection{Step 1: Adaptive Reference Windowing}

EADD maintains two windows: a \textit{reference window} $W_{ref}$ ($N_{ref}$ samples) and a \textit{current window} $W_{cur}$ ($N_{cur}$ samples). The reference window is updated by \textit{reservoir sampling}~[22]: each new sample $x_t$ replaces a random element in $W_{ref}$ with probability $N_{ref}/t$, which keeps an unbiased sample of historical data over long streams. The current window is a standard sliding buffer of recent observations. Monitoring is triggered every $M$ samples to balance latency and cost; after confirmed drift, $W_{ref}$ is reset to the new regime.

\subsection{Step 2: Discriminative Classification and Prediction Entropy}

Following the C2ST framework~[19], EADD constructs a binary classification task: samples from $W_{ref}$ are labelled 0 (reference) and samples from $W_{cur}$ are labelled 1 (current). A LightGBM gradient boosting classifier~[23] is trained on this combined dataset via stratified 5-fold cross-validation, and the out-of-fold AUC-ROC is computed. AUC~$\approx$~0.5 indicates the classifier cannot distinguish the two windows (no drift); AUC~$\gg$~0.5 indicates the distributions are separable (drift suspected). We use LightGBM rather than D3's logistic regression~[9] because gradient boosting captures non-linear feature interactions and has been shown to be a more powerful discriminator in two-sample testing~[19].

\textbf{Prediction Entropy.} Beyond the aggregate AUC score, EADD extracts additional diagnostic information from the classifier's per-sample predictions. We compute the normalised binary entropy of the discriminative classifier's predicted probabilities:
\begin{equation}
H_{\text{pred}} = \frac{-1}{n}\sum_{i=1}^{n}\left[\hat{p}_i\log\hat{p}_i + (1{-}\hat{p}_i)\log(1{-}\hat{p}_i)\right]
\label{eq:pui}
\end{equation}
where $\hat{p}_i = f_\theta(x_i)$ is the predicted probability that sample $i$ belongs to $W_{cur}$. $H_{\text{pred}} \approx \ln 2$ indicates maximum uncertainty (no drift); $H_{\text{pred}} \rightarrow 0$ indicates high classifier confidence (drift likely). A declining $H_{\text{pred}}$ trend across consecutive windows serves as an early-warning signal for emerging drift before statistical significance is reached.

\subsection{Step 3: Adaptive Sequential Permutation Test (ASPT)}

If AUC exceeds a threshold $\tau$, EADD applies an \textbf{adaptive sequential permutation test} inspired by Gandy~[30]. Instead of a fixed budget of $B$ permutations, ASPT monitors evidence accumulation and terminates early:
\begin{enumerate}
    \item For each permutation $i = 1, \ldots, B_{\max}$: shuffle labels, retrain, compute $\text{AUC}^{(i)}_{\text{perm}}$. Update count $c_i = \#\{j \leq i : \text{AUC}^{(j)}_{\text{perm}} \geq \text{AUC}_{\text{actual}}\}$.
    \item \textbf{Early reject $H_0$:} If $c_i = 0$, $i \geq B_{\min}$, and $1/(i+1) < \alpha$, terminate and confirm drift with $p = 1/(i+1)$.
    \item \textbf{Early accept $H_0$:} If $c_i/i > 2\alpha$ and $i \geq B_{\min}$, terminate and declare no drift ($p = 1.0$).
    \item \textbf{Terminal:} If $B_{\max}$ is reached, $p = (c_{B_{\max}}{+}1)/(B_{\max}{+}1)$.
\end{enumerate}
Drift is confirmed if $p < \alpha$. This preserves standard permutation-test interpretation under adaptive stopping by using either the terminal estimate or the conservative early-reject upper bound.

\subsection{Step 4: Feature Attribution and Drift Severity Index}

Upon drift confirmation, EADD exploits the trained discriminative classifier---which D3~[9] would discard---to diagnose \textit{which} features drive the detected drift using SHAP (SHapley Additive exPlanations)~[24].

\textbf{Feature Attribution.} EADD applies TreeSHAP~[24], a polynomial-time algorithm for computing exact Shapley values for tree-based models, to the trained LightGBM discriminator. For each sample, TreeSHAP produces a SHAP value $\phi_j$ for each feature $j \in \{1, \ldots, d\}$, indicating how much that feature contributed to the classifier's ability to distinguish reference from current data. \textit{Per-feature} importances (each feature's individual contribution, averaged across all samples) are computed as $\bar{\phi}_j = \frac{1}{n}\sum_{i=1}^{n}|\phi_j^{(i)}|$ and normalised into a contribution vector $\mathbf{s} = (s_1, \ldots, s_d)$ where $s_j = \bar{\phi}_j / \sum_k \bar{\phi}_k$. This vector $\mathbf{s}$ serves two purposes: (1)~it identifies which individual features drive the detected drift, and (2)~it feeds directly into the DSI computation below to quantify drift severity. Additionally, EADD computes \textbf{SHAP interaction values}~[31] via the interaction matrix $\Phi \in \mathbb{R}^{d \times d}$, where $\Phi_{ij}$ captures the joint effect of features $i$ and $j$ beyond their individual contributions. Significant interactions (exceeding $\mu + 2\sigma$ of the interaction distribution) are reported as supplementary diagnostics, addressing \textit{correlated feature drift}---where the joint distribution $P(X_i, X_j)$ shifts while individual feature distributions remain stable---a pattern that per-feature attribution alone misses~[32]. The per-feature contribution vector $\mathbf{s}$ is used for DSI severity quantification, while interaction diagnostics guide practitioners toward correlated root causes.

\textbf{Drift Severity Index (DSI).} The normalised contribution vector $\mathbf{s}$ computed from TreeSHAP directly feeds the DSI: a composite severity metric combining detection confidence with attribution concentration:
\begin{equation}
\text{DSI} = \text{AUC} \times \left(1 - H_{\text{norm}}(\mathbf{s})\right)
\label{eq:dsi}
\end{equation}
where $\mathbf{s} = (s_1, \ldots, s_d)$ is the normalised SHAP contribution vector (i.e., $s_j = \bar{\phi}_j / \sum_k \bar{\phi}_k$), $d$ is the number of input features, and $H_{\text{norm}}(\mathbf{s}) = -\sum_{j=1}^{d} s_j \log_2 s_j \,/\, \log_2 d$ is the normalised Shannon entropy~[34]. The $\log_2 d$ denominator ensures $H_{\text{norm}} \in [0, 1]$ regardless of dimensionality. Intuitively, when drift is concentrated in a single feature, $H_{\text{norm}} \approx 0$ and $\text{DSI} \approx \text{AUC}$ (high severity); when drift is uniformly spread across all features, $H_{\text{norm}} \approx 1$ and $\text{DSI} \approx 0$ (diffuse shift). DSI-based prescriptions replace heuristic thresholds:
\begin{itemize}
    \item \textbf{Critical (DSI $> 0.6$):} Concentrated drift $\rightarrow$ audit specific data pipeline.
    \item \textbf{Moderate ($0.3$--$0.6$):} Subset drift $\rightarrow$ investigate shared data source.
    \item \textbf{Low ($< 0.3$):} Diffuse shift $\rightarrow$ schedule full model retraining.
\end{itemize}

\subsection{Algorithm Summary}

\begin{algorithm}[htbp]
\caption{EADD Detection Cycle}
\label{alg:eadd}
\begin{algorithmic}[1]
\REQUIRE Stream $\{x_t\}$, $W_{ref}$, $W_{cur}$, threshold $\tau$, significance $\alpha$
\ENSURE Drift decision, $H_{\text{pred}}$, DSI, feature attribution, prescription
\STATE Update $W_{cur}$; update $W_{ref}$ via reservoir sampling~[22] (each new $x_t$ replaces a random element with probability $N_{ref}/t$, maintaining a uniform sample of history)
\STATE Train LightGBM on $W_{ref} \cup W_{cur}$ with labels $\{0,1\}$
\STATE Compute AUC via stratified 5-fold CV
\STATE Compute $H_{\text{pred}}$ via Eq.~(\ref{eq:pui})
\IF{AUC $> \tau$}
    \STATE $p \gets$ ASPT($W_{ref}$, $W_{cur}$, AUC) \COMMENT{Adaptive early stopping}
    \IF{$p < \alpha$}
        \STATE Compute SHAP values + interaction matrix via TreeSHAP
        \STATE Compute DSI via Eq.~(\ref{eq:dsi})
        \STATE Generate DSI-based prescription
        \RETURN \textsc{Drift}, $H_{\text{pred}}$, DSI, SHAP ranking, prescription
    \ENDIF
\ENDIF
\RETURN \textsc{No-Drift}, $H_{\text{pred}}$
\end{algorithmic}
\end{algorithm}

% ===== EXPERIMENTAL EVALUATION =====
\section{Experimental Evaluation}

\subsection{Experimental Design}

Three experiments were designed to validate EADD's detection accuracy, feature attribution, and computational efficiency. Following the evaluation protocol of Lukats et al.~[10], EADD was benchmarked against seven state-of-the-art unsupervised detectors: D3~[9], BNDM, CSDDM~[15], IBDD~[16], OCDD~[17], SPLL~[18], and UDetect.

\textbf{Experiment 1 (Synthetic Data):} Synthetic streams ($n{=}5{,}000$, $d{=}5$) with controlled abrupt, gradual, incremental, and recurring drift. All eight detectors were compared on each drift type. Additionally, false alarm robustness was evaluated on four stable synthetic streams (Gaussian i.i.d., autocorrelated AR(1) with $\phi{=}0.8$, heteroscedastic, correlated with $\rho{=}0.7$) containing zero drift, with five independent runs per stream type.

\textbf{Experiment 2 (Real-World Benchmark):} 11 real-world datasets from the USP Data Stream Repository~[25] following the Lukats et al.~[10] evaluation protocol: datasets with known drift points are evaluated by Mean Time to Detection (MTD); datasets without ground truth are evaluated by total number of detections (selectivity).

\textbf{Experiment 3 (SHAP Attribution Accuracy):} Three controlled synthetic scenarios ($n{=}10{,}000$, $d{=}10$ features, drift injected at $t{=}5{,}000$): (a)~univariate drift in F3 only (mean shift $+2\sigma$), (b)~subset drift in F2/F5/F7 simultaneously, (c)~multivariate drift in all features. Precision@k (P@k) and Recall@k (R@k) measure whether the top-$k$ SHAP-ranked features exactly match the $k$ truly drifted features.

D3~[9]: logistic regression (liblinear solver), matching window sizes, fixed AUC threshold 0.7. All experiments: Python 3.10, LightGBM 4.1.0, SHAP~[24] 0.42.1, scikit-learn 1.3.2. Random seeds fixed for reproducibility.

% ===== CONCLUSION =====
\section{Conclusion}

This paper presented EADD (Enhanced Adversarial Drift Detection), a framework that extends discriminative drift detection with three contributions: (1) an Adaptive Sequential Permutation Test with prediction entropy monitoring for statistically rigorous drift confirmation, (2) SHAP interaction-aware feature attribution and a Drift Severity Index for feature-level diagnosis and severity quantification, and (3) comprehensive experimental evaluation against seven state-of-the-art detectors.

Evaluation demonstrated that EADD is the only detector to achieve full drift-type coverage (4/4) with zero false alarms on synthetic benchmarks, detects drift on three of four annotated real-world datasets with competitive delay, achieves perfect feature attribution accuracy (P@k\,{=}\,R@k\,{=}\,1.0), and produces statistically significantly fewer false alarms than D3 ($p = 0.0101$). By answering ``Is drift emerging?'', ``Is it statistically significant?'', ``Which features caused it?'', and ``How severe is the shift?'', EADD transforms binary drift alarms into actionable diagnostic reports.

\subsection{Limitations and Future Work}

ASPT reduces computational cost by over 75\% compared to fixed-budget permutation testing, but the adversarial paradigm remains ${\sim}50\times$ costlier than threshold-based D3. The DSI severity tiers ($0.3$, $0.6$) require domain-specific calibration. SHAP attributions are correlational, not causal~[33]. Future work includes e-value sequential testing~[27], causal drift attribution~[33], and adversarial robustness evaluation.
