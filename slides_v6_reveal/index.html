01Distribution drift

A feature quietly shifts

A model assumes tomorrow looks like training. It rarely does — inputs drift, and accuracy decays silently.

A classifier two-sample test catches it: if a classifier can separate reference from current, the distribution moved.

02The discriminative paradigm

Drift = a classification problem

C2ST · Classifier Two-Sample Test [3]
Train a classifier to tell reference from current data. If it succeeds (AUC ≫ 0.5), the two distributions differ → drift.
AUC
Area under the ROC curve — separability. 0.5 = indistinguishable, 1.0 = perfectly separable.

D3[1] brought C2ST to streams; ranked the most robust unsupervised detector[2] — but is linear, fixed-threshold, and discards the model.

02Research gap

Three gaps in D3

1

No significance test

Fixed AUC=0.7 → false-alarm storms on autocorrelated data.[1]

2

No attribution

“Detect-and-discard” — never says which feature drifted. Prior attempts are supervised or batch-only.[1][13][14]

3

Linear only

Misses non-linear, gradual & incremental drift.[1][2]

No prior work integrates significance + attribution + severity in one unsupervised streaming pipeline.

03Step 1 · design choice

Reservoir windows + non-linear C2ST

Reservoir reference

Reservoir sampling8 keeps a representative reference window over the whole stream — not just the last block.

LightGBM detector

A non-linear gradient-boosted classifier5 captures interactions a linear D3 misses → broader coverage.

why enables all three contributions; coverage 4/4 vs D3’s 2/4 comes from here.

03Contribution 1a

Prediction entropy H_pred

Entropy [9]
A measure of uncertainty. High = the classifier is unsure (no drift); low = it is confident it can separate them (drift building).
H_pred = −Σ pᵢ log₂ pᵢ

An early-warning signal on every step — even before AUC crosses the threshold.

03Contribution 1b

ASPT — stop permuting once it’s decisive

Permutation test
Shuffle the ref/current labels many times. If the real split is rarely beaten by chance, the drift is statistically significant.
ASPT · adaptive early stop [4]
Stop as soon as the evidence is decisive — instead of always running all 199.

90–95% fewer permutations · Type-I error preserved

03Contribution 2a

SHAP localisation — which features?

SHAP [6][7]
Splits the model's decision into each feature's contribution. The biggest share = the feature that drove the drift.

D3 throws its classifier away. EADD keeps it and reads the explanation — a detector becomes a diagnostic.

In the test case the true feature F3 is ranked #1 — P@k = R@k = 1.0.

03Contribution 2b

Drift Severity Index

DSI = AUC × (1 − H_norm(s))
DSI · Drift Severity Index
Combines how sure we are (AUC) with how concentrated the blame is (low SHAP entropy). One score, 0 → 1.
Low <0.3diffuse → scheduled retrain
Moderate 0.3–0.6subset → audit shared source
Critical >0.6concentrated → audit pipeline

heuristic tiers; require domain calibration.

Honest limits — and where this goes next.
05Limitations

What EADD cannot claim — yet

  • Wall-clock ~50–160× D3 — LightGBM retrains every M=50 steps.
  • Multiple testing over time: ~600 tests/30k samples; zero-FA result is empirical, not theoretically guaranteed.
  • DSI tier boundaries are heuristic — never triggered above LOW in experiments.
  • SHAP attributions are correlational, not causal.
05Future work

Each gap is a thread

  • Anytime-valid e-values for streaming FWER control.
  • Causal attribution (Budhathoki+ 202110).
  • Multi-domain DSI calibration.
  • Adversarial robustness of discriminative detectors.
